# Linee di esecuzione multiple

In questo caso abbiamo più linee di esecuzione contemporaneamente, una per ciascun thread (<span class="hl-orange">main-thread</span> e <span class="hl-purple">thread-1</span>):

```{.python linenums="1" data-which-purple="4" data-which-orange="11"}
import threading, from time import sleep

def thread_1():
    while True:
        print("Dentro il thread")
        sleep(1)

t1 = threading.Thread(target=thread_1)
t1.start()

while True:
    print("Dentro il main-thread....")
    sleep(1)
```

!!! danger "Prevedibilità della prossima linea di esecuzione"
    Quando si hanno più thread, e quindi più linee di esecuzione, {==^^diventa IMPOSSIBILE prevedere la PROSSIMA linea di esecuzione^^==}.

La scelta dipende dalla libreria di threading, la quale:

- Permetterà l'esecuzione a ciascun thread.
- ^^**Ma** non permetterà di prevedere quale sarà la prossima linea di esecuzione.^^

- Il <span class="hl-orange">main-thread</span> deve eseguire la linea #11.
- Il <span class="hl-purple">thread-1</span> deve eseguire la linea #4.

## Sequenza di esecuzione: linea 1

Facciamo il caso che <span class="hl-purple">thread-1</span> acquiserà l'utilizzo della CPU eseguendo la linea #5.
Allora:

- La linea di esecuzione di <span class="hl-purple">thread-1</span> ^^avanzerà^^.
- La linea di esecuzione di <span class="hl-orange">main-thread</span> rimane ferma.


```{.python linenums="1" data-next-purple="4" data-still-orange="11"}
import threading, from time import sleep

def thread_1():
    while True:
        print("Dentro il thread")
        sleep(1)

t1 = threading.Thread(target=thread_1)
t1.start()

while True:
    print("Dentro il main-thread....")
    sleep(1)
```
