
# La linea di esecuzione

Per capire i thread dobbiamo aver ben chiaro quella che chiameremo la ==linea di esecuzione==, ovvero la prossima linea di codice che verrà eseguita.

!!! warning Importante

    In queste dispense, la linea di esecuzione sarà quella evidenziata.

---

Analizziamo la sequenza della ==linea di esecuzione== del seguente script python:

```python linenums="1"
nome=input("Inserisci il tuo nome")
if len(nome) < 5:
    print("Nome troppo corto.")
else:
    print(f"Ciao {nome}")
```

Per due diversi input:

- `"Francesco"`, con lunghezza maggiore di 5.
- `"Anna"`, con lunghezza minore di 5.


!!! warning "Attenzione"

    1. La linea evidenziata sarà la prossima linea ad essere eseguita.
    2. Naturalmente, la linea evidenziata non è stata ancora eseguita.
    3. Solo una linea può essere evidenziata.
    4. Le linee vuote vengono ignorate.

---

!!! note "Input"

    `Francesco`

Il programma parte dalla linea #1:

```python linenums="1" hl_lines="1"
nome=input("Inserisci il tuo nome")
if len(nome) < 5:
    print("Nome troppo corto.")
else:
    print(f"Ciao {nome}")
```

Eseguita la linea #1 la linea di esecuzione avanza alla linea #2:

```python linenums="1" hl_lines="2"
nome=input("Inserisci il tuo nome")
if len(nome) < 5:
    print("Nome troppo corto.")
else:
    print(f"Ciao {nome}")
```

Eseguita la linea #2 la linea di esecuzione salta alla linea #5:

```python linenums="1" hl_lines="5"
nome=input("Inserisci il tuo nome")
if len(nome) < 5:
    print("Nome troppo corto.")
else:
    print(f"Ciao {nome}")
```

!!! abstract "Output"

    `Ciao Francesco`


Quindi la sequenza è stata:

- Linea #1
- Linea #2
- Linea #5
