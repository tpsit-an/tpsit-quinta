
# La prossima linea di esecuzione è prevedibile

Nell'esempio precedente:

```python linenums="1"
nome=input("Inserisci il tuo nome")
if len(nome) < 5:
    print("Nome troppo corto.")
else:
    print(f"Ciao {nome}")
```

abbiamo già affermato che, se è noto l'input, **^^sappiamo con certezza^^** quale sarà la sequenza della linea di esecuzione.

Inoltre riguardo a questo script, possiamo affermare che la prossima linea sarà **^^sicuramente sotto^^** quella attuale.

Questo non è sempre vero, ad esempio nei loop:

```python linenums="1" hl_lines="1"
nome=""
while len(nome) < 5:
    nome=input("Inserisci il tuo nome")
print(f"Ciao {nome}")
```

Si entra nel `while` dato che `nome` ha lunghezza zero:

```python linenums="1" hl_lines="2"
nome=""
while len(nome) < 5:
    nome=input("Inserisci il tuo nome")
print(f"Ciao {nome}")
```

Procede alla richiesta dell'input:

```python linenums="1" hl_lines="3"
nome=""
while len(nome) < 5:
    nome=input("Inserisci il tuo nome")
print(f"Ciao {nome}")
```

Riparte dalla linea #2, ovvero **^^torna sù^^**:

```python linenums="1" hl_lines="2"
nome=""
while len(nome) < 5:
    nome=input("Inserisci il tuo nome")
print(f"Ciao {nome}")
```

In ogni caso, loop o non loop e noto l'input, la sequenza della linea di esecuzione è {++SEMPRE PREVEDEBILE al 100%++}
