# `connect` e `accept`

Nel protocollo TCP, la connessione viene stabilita tramite un **three-way handshake** (stretta di mano a tre vie):

1. `client` ÔåÆ `server` - pacchetto  **SYN** (*synchronize*):
    - Il client richiede l'apertura della connessione dicendo:
    > client: _"il mio numero di sequenza ├¿ X"_.

2. `server` ÔåÆ `client` - pacchetto **SYN+ACK** (*synchronize-acknowledge*):
    - Il server conferma al client inviandogli un pacchetto che ├¿ ==^^sia^^ SYN che ACK==:
    > server: _"il tuo numero di sequenza ├¿ X, il mio numero di sequenza ├¿ Y"_.

3. `client` ÔåÆ `server` - pacchetto **ACK** (*acknowledge*):
    - Il client conferma la ricezione del **SYN+ACK** del server:
    > client: _"il tuo numero di sequenza ├¿ Y"_.

4. A questo punto la connessione ├¿ stabilita e i dati possono essere trasmessi.

## Diagramma di Gantt

Di seguito il diagramma di Gantt dell'utilizzo della CPU durante `connect()` e `accept()`:

### Client - `connect()`

Quando viene eseguita la funzione `connect()` la linea di esecuzione ├¿ bloccata. Durante tale esecuzione, la CPU viene utilizzata **due** volte:

- Quando viene avviata:
    - La CPU viene usata per inviare il pacchetto `SYN` al server.
- Quando arriva il pacchetto `SYN+ACK` dal server:
    - La CPU viene usata per elaborare il pacchetto.
    - La CPU viene usata per inviare l'`ACK` al client.

Succesivamente la linea di esecuzione ├¿ sbloccata.

![Gantt connect](img/gantt_connect.svg)

### Server - `accept()`

Quando viene eseguita la funzione `accept()` la linea di esecuzione ├¿ bloccata. Durante tale esecuzione, la CPU viene utilizzata **tre** volte:

- Quando viene avviata:
    - La CPU viene usata per mettersi in _posizione_ di ascolto di connessioni in entrata.
- Quando arriva il pacchetto `SYN`$_53$ dal client:
    - La CPU viene usata per elaborare il pacchetto.
    - La CPU viene usata per inviare l'`SYN+ACK` al client.
- Quando arriva l'`ACK` finale dal client:
    - La CPU viene usata per elaborare il pacchetto.
Succesivamente la linea di esecuzione ├¿ sbloccata.
